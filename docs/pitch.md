# Pitch de 3 minutos

## Hook

Wattless no es solo un auditor; es un reductor de entropía digital. Aplicamos los principios de la termodinámica al desarrollo web para que el software deje de calentar el planeta innecesariamente.

## Problema

La web tiene un coste invisible. Cada imagen pesada, cada tercero innecesario y cada script bloqueante consumen energía, generan calor residual y empeoran la experiencia de usuario.

En otras palabras: más bytes no solo significan páginas más lentas, también significan más coste operativo y más ineficiencia.

## Producto

Wattless escanea una URL real y conecta en un solo informe:

- transferencia observada en runtime
- `CO2` estimado por visita
- `LCP`, `FCP` y tiempo de scripts
- elementos vampiro con contexto visual
- acciones prioritarias generadas con IA
- un `Green Fix` orientado a un snippet real

## Demo

1. Pegamos una URL real.
2. Wattless calcula bytes, CO2, `LCP` y dependencia de terceros.
3. El dashboard muestra qué recurso concentra más coste y por qué.
4. Pegamos un snippet real y obtenemos una propuesta de refactor más ligera.
5. Exportamos el informe como Markdown listo para README o PR.

## Cierre

Menos bytes significan menos calor, menos coste y mejor UX. Wattless convierte esa intuición en una herramienta técnica, visual y accionable para equipos que quieren construir una web más sostenible.

## Respuesta corta si preguntan por la física

Sí, el software también calienta el planeta. Ningún proceso es 100% eficiente: parte de la energía eléctrica termina disipándose como calor. En la web eso ocurre tanto en servidores como en el dispositivo de la persona usuaria. Si enviamos más datos de los necesarios o bloqueamos CPU con JavaScript inútil, estamos gastando energía sin aportar valor. Wattless ataca exactamente esa ineficiencia.
